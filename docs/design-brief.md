Design Brief
ScenicAI – AI-Powered Scenic Walk Planner
1. Project Overview

ScenicAI is a web application that generates personalized scenic walking routes using an AI agent.
Users enter their location, desired walk duration, and aesthetic preferences (nature, water, historic, quiet), and the system returns an optimized scenic route with an explanation.

The interface should feel:

Calm

Intelligent

Trustworthy

Nature-inspired

Minimal but modern

2. Design Goals

The design must:

Make route planning feel effortless

Clearly communicate “AI-powered intelligence”

Highlight scenic quality visually

Keep the map as the main focus

Support conversational refinement

3. Target Audience

Students and young professionals

Urban walkers

Tourists

Tech-savvy users

Recruiters viewing the demo

Tone: modern, clean, thoughtful.

4. Core Screens
1️⃣ Landing / Main Interface

Elements:

Map (center focus)

Duration slider

Preference toggles (Nature / Water / Historic / Quiet)

“Generate Scenic Route” button

Small AI explanation panel

The map should occupy 60–70% of screen space.

2️⃣ Generated Route View

Highlighted scenic path

Scenic score indicator (e.g., 82/100)

Short AI explanation:

“This route follows river paths and passes through two parks while avoiding major roads.”

“Refine” input box (conversational adjustment)

3️⃣ Conversational Refinement

Chat-style mini interface:

User: “Make it shorter”

AI: updates route and explains changes

Should feel lightweight — not full chatbot UI.

5. Visual Direction
Mood

Calm

Nature-inspired

Soft gradients

Subtle motion

Clean typography

Avoid:

Harsh neon colors

Heavy dashboard feel

Overly technical UI

6. Color Palette

Primary:

Soft green (nature trust)

Muted blue (water)

Warm sand/beige (grounded calm)

Accent:

Subtle gradient for scenic score indicator

Background:

Light neutral or very soft off-white

7. Typography

Modern sans-serif

Clean and readable

Slightly rounded feel

Avoid overly corporate fonts

Hierarchy:

Large map-focused layout

Clear section separation

Minimal clutter

8. Key UI Components

🌍 Interactive map (primary visual anchor)

🎚 Duration slider

🧭 Preference toggles (pill buttons)

⭐ Scenic score badge

💬 AI explanation card

🔄 Regenerate / Refine button

9. Interaction Principles

Fast feedback

Subtle loading animation (route drawing animation)

Smooth map transitions

Clear visual difference between alternative routes

Explanations appear naturally after route renders

10. Emotional Outcome

When the user finishes using ScenicAI, they should feel:

“This understands what I want.”

“This feels smarter than Google Maps.”

“This is peaceful and intentional.”
